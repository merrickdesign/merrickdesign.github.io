Heroic: Product development overview

Problem

Heroic, a PVP strategic card batter, had been stuck in development for two years. In the role of UX designer as part of a cross-functional team, we used design best practices to push the game to full launch.

My Role

As the sole UX Designer on the team, I coached the game designers and artists through the design thinking process. Ran ideation, worked with marketing and research, created prototypes, design guidelines and system.

Approach

Taking the game to global launch was a detailed process involving breaking the game down to its core experience, validating the value prop, identifying users then building the experience back up. This is an overview of the high-level phases and activities that delivered the project.

Validate value prop and Audience

I worked with the user research and marketing team to validate our value proposition and identify our audience. From there, we created personas which were used as reference points on which we could hang the experience.

Build an experience

We built a user journey with for a single game loop, capturing the steps and user thoughts or questions.

We expanded this map to encompass the full user journey detailing phases; attract, enter, engage, exit, extend. Adding in details such as the functionality needed at each step, this defined the functionality when designing screens.

Placeholders

Having defined steps and functionality, I created basic layouts for each screen. Adding them to the board as placeholders.

User testing

As we tested the experience with players we identified problems and critical areas to solve. I ran ideation sessions to explore solutions and worked with user researchers to test the solutions with our audience.
